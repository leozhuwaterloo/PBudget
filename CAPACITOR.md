# PlaidBudget — native apps (Capacitor)

The iOS/Android apps are a **thin Capacitor shell** whose webview loads the live
site (`https://pbudget.ppvnx.com`, set in `capacitor.config.ts` → `server.url`).
There is nothing to bundle: the app is server-rendered with cookie auth and
Postgres. **Consequence: every deploy of `pbudget.ppvnx.com` ships an app update
instantly.** You only rebuild/resubmit the native binary when the icon, splash,
Capacitor version, native plugins, or store metadata change — not for app logic.

- **Bundle ID (permanent, both stores):** `com.ppvnx.pbudget`
- **Display name:** PlaidBudget
- Native projects (`android/`, `ios/`) plus `capacitor-www/`, `resources/`, and
  `capacitor.config.ts` are excluded from the Docker build via `.dockerignore`,
  so they never enter the k3s image — the web deploy is unaffected.

## Regenerating assets

Icon + splash source art lives in `resources/` (generated by
`scripts/gen-app-icon.mjs` from the brand palette — ledger green `#15684a` on
cool paper `#e9ebe5`, matching `src/app/globals.css`). To rebuild every density:

```bash
nvm use 22
node scripts/gen-app-icon.mjs           # only if the art/palette changed
npx capacitor-assets generate           # --android / --ios to scope
```

`resources/icon.png` is a placeholder (white serif "P" on the ledger-green
field). Drop in real 1024² brand art at that path and re-run the two commands
above to replace it.

## Android (buildable on this Linux box)

SDK at `~/Android/sdk`, JDK 21 at `~/Android/jdk21`. `local.properties` (sdk.dir)
is gitignored — recreate it on a fresh checkout.

```bash
cd android
export ANDROID_HOME=$HOME/Android/sdk JAVA_HOME=$HOME/Android/jdk21
printf 'sdk.dir=%s\n' "$ANDROID_HOME" > local.properties

./gradlew assembleDebug     # → app/build/outputs/apk/debug/app-debug.apk  (sideload/test)
./gradlew bundleRelease     # → app/build/outputs/bundle/release/*.aab      (Play upload)
```

**Release signing (once).** Play requires a signed AAB. Generate an upload
keystore, keep it **off git and backed up** (losing it means you can't update the
app):

```bash
keytool -genkey -v -keystore pbudget-upload.jks -alias upload \
  -keyalg RSA -keysize 2048 -validity 10000
```

Then add a `signingConfigs` block to `android/app/build.gradle` reading the
keystore path/passwords from `~/.gradle/gradle.properties` (never commit those),
and wire it into `buildTypes.release`. Upload the `.aab` in the Play Console →
create the app with package name `com.ppvnx.pbudget`.

## iOS (Mac + Xcode, but no persistent checkout)

`scripts/ios-release.sh` builds and uploads to App Store Connect from an
**ephemeral clone** — the repo is cloned to a temp dir and deleted on exit, so
nothing (no `ios/` folder, no checkout) stays on your Mac. Keep just that one
script (e.g. `~/bin/`) plus a `~/.pbudget-ios.env` secrets file.

**One-time setup:**
1. Prereqs: Xcode + `xcode-select --install`, `brew install cocoapods`, Node 18+.
2. App Store Connect API key: Users and Access → Integrations → App Store Connect
   API → generate a key with the **App Manager** role. Download the `.p8` once,
   note the Key ID + Issuer ID.
3. Create the app record once: App Store Connect → Apps → **+** → New App, bundle
   id `com.ppvnx.pbudget`. (Automatic signing registers the bundle id itself;
   this step is only the store listing.)
4. `cp scripts/ios-release.env.example ~/.pbudget-ios.env`, fill in the key
   IDs / Team ID, `chmod 600` it. **Never commit it.**

**Every release:**
```bash
~/bin/ios-release.sh    # clone → build → sign → upload → self-clean
```
The build appears in App Store Connect → TestFlight in a few minutes; submit for
review from there. Build number is an epoch timestamp (auto-unique per upload);
override the user-facing version with `PB_MARKETING_VERSION`.

Prefer the GUI once? `npx cap add ios && npx cap open ios` in a manual clone still
works — but that leaves the checkout on your Mac, which the script avoids.

## The sync workflow

After changing `capacitor.config.ts`, adding a plugin, or regenerating assets:

```bash
npx cap sync        # copies config + plugins into android/ and ios/
```

App *logic* changes need no sync — they go live via the normal server deploy.

## Store review notes

- **Apple Guideline 4.2 (minimum functionality)** rejects apps that are just a
  website in a shell. PlaidBudget clears this on substance — Plaid bank-account
  linking, automatic transaction import + categorization, budgets, and per-user
  accounts are real app functionality, not a brochure. If review still pushes
  back, the strongest hedge is adding **push notifications**
  (`@capacitor/push-notifications`) for budget alerts / new-transaction pings —
  genuinely useful here and the canonical 4.2 fix. Google Play is more lenient
  but has the same "no thin webview spam" spirit.
- Both stores need: privacy policy URL, screenshots (phone + tablet / iPad),
  a data-collection disclosure (we collect email + auth cookie, and via Plaid
  bank-transaction data), and an age rating. iOS also needs an
  `NSAppTransportSecurity`-clean HTTPS origin — ours is HTTPS-only, so nothing
  to configure.
- Because the app links financial accounts through Plaid, be ready to explain
  the Plaid data flow in App Review notes and disclose financial-data collection
  in both stores' privacy questionnaires.
